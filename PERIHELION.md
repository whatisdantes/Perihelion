# Перигелий (Perihelion) — заметки проекта / хендофф

> Документ для продолжения работы (в т.ч. в новой сессии Claude на другом устройстве).
> Последнее обновление: 2026-06-16.

## Что это

Проект начинался как ванильный (без сборщика) **Three.js r161** 3D-макет Солнечной системы
(`index.html`) + чёрная дыра Гаргантюа из «Интерстеллара» (`blackhole.html`), связанные
переходом через червоточину. Сейчас превращаем его в аркадную игру-исследование
**«Перигелий»** в духе No Man's Sky / EVE Online.

**Игровая петля:** лететь → сканировать → открыть (кодекс + награда) → апгрейд → варп.
**Идентичность:** курируемая, астрономически РЕАЛЬНАЯ система (не бесконечная процедурка) +
реальный EVE-масштаб + кинематографичная чёрная дыра как финал.

## Сделано (закоммичено, ветка `master`)

- **Полётная модель** (`js/flight.js`): кватернионное 6DOF-управление, чейз-камера, буст
  (рывок FOV), мышь через pointer-lock (с клэмпом и `unadjustedMovement`), **аркадный
  дрейф + тормоз** (Space — тормоз). Вход/выход — клавиша **F**.
- HUD полёта + центральный прицел.
- **Сканирование + кодекс**: захват цели в конусе (залок на время скана, чтобы соседи не
  перебивали), скан по ЛКМ/R (~0.6с) → открытие: +100 кредитов, баннер, ✓ в сайдбаре,
  сохранение в `localStorage`; лор объекта (desc/facts) открывается только после скана.
- **Реальный EVE-масштаб, этапы 1–4 из 8** (см. ниже) — полёт к Земле в натуральную
  величину уже работает без дрожания.

## Ключевые технические решения (реальный масштаб)

- **Единица: 1 ед. сцены = 1000 км. 1 а.е. = 149 600 ед.** Единый честный масштаб
  размеров И расстояний, без сжатия. `newRadius = радиус_км/1000`,
  `newDist = а.е.×149600` (луны — км/1000). Солнце r696, Земля r6.371 @149600,
  Нептун @4.5M, Вояджер @25M. Скорость в ед/с = км/с.
- Данные в `js/data.js`: `SCALE`, флаг `REALISTIC` (сейчас **true**), таблица `REAL`,
  применяется `applyRealisticScale()`. Радиусы крошечных аппаратов/лун чуть приподняты для
  кликабельности.
- **Floating-origin обязателен** (сам масштаб Float32 не чинит). В `js/main.js` у каждого
  тела `rec.wpos` (Float64); `applyOrigin()` — единственная точка Float64→Float32
  (`group.position = wpos - origin`). Полёт: `origin = shipCtl.wpos` (корабль в нуле, мир
  движется вокруг). Обзор: `origin = 0` (как раньше). `shipCtl.wpos` в `js/flight.js`.
- Шейдеры больше не считают Солнце в начале координат: общий uniform `uSunLocal`
  (сдвинутая позиция Солнца) для атмосферы, аналитических теней и Земли; `sunLight.position`
  следует за Солнцем; `uSunDir` берётся из wpos.
- **Глубина: `logarithmicDepthBuffer: true`** (в r161 НЕТ `reversedDepthBuffer`). Проверено:
  все кастомные шейдеры (Солнце, Земля, атмосфера, сияния) + 4xMSAA + bloom работают.
- Камера/лимиты/обзор зависят от флага `REALISTIC` (far 3e7, OrbitControls max 5e7).
  Звёздный фон/туманности/Млечный Путь — в `skyGroup`, следующем за камерой («бесконечен»).

## Известные «сырые» места (на этапы 5–8)

- Брекетов нет → планеты издалека невидимы (этап 5, следующий).
- Планеты быстро орбитят в абсолютных ед. при 1ч/с (Земля ~30 ед/с) — сублайтом не догнать;
  нужен варп + более медленный темп времени по умолчанию.
- Полутень затмений/тени колец использует старый радиус Солнца `16.0` и `uMoonR 0.55`.
- `scanRangeOf`, базовые размеры колец выделения/скана — под старый масштаб.
- Сублайт корабля maxSpeed 60 / accel 40 — временные значения под новый масштаб.

## Как запускать и проверять

- **Дев-сервер:** ES-модули без сборщика требуют http (не file://). Запуск через превью-конфиг
  **`solar-system`** в `.claude/launch.json` → она запускает **`devserver.py`**
  (многопоточный, отдаёт `Cache-Control: no-store` на `127.0.0.1:8137`). **No-cache важен** —
  обычный `python -m http.server` кэшировал ES-модули, и правки не подхватывались. Не
  возвращать к нему.
- **Особенность превью:** headless-вкладка превью усыпляет `requestAnimationFrame` в фоне →
  `preview_screenshot` виснет, рендер замирает (логика/eval при этом работают). **Перезапуск
  превью-сервера (stop+start) будит рендер.**
- **Отладочные хуки:** `window.__app = { renderer, composer, scene, camera, bodies, controls,
  shipCtl, playerShip, toggleShipMode, state, startScan, completeScan, persist, resetSave,
  origin, applyOrigin, updateBodies, uSunLocal }`. `window.__cmeEarth()` — выброс к Земле.
  Тюнинг полёта вживую: `window.__app.shipCtl` (mouseSens, maxSpeed, driftDamping,
  brakeDamping, maxTurn, camBack/camUp/camLead). `resetSave()` чистит сейв.
- **Сейв:** `localStorage['perihelion.save.v1']` = `{v, discovered:[ids], credits}`.
- **Git:** ветка `master`. Коммитить каждый этап отдельно; в конце сообщения —
  трейлер `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`. Базовый до-игровой
  коммит есть для отката.

## Предпочтения пользователя

- Объём: отшлифовать компактный MVP-срез (не бесконечные фичи).
- Тон: **аркадный полёт** — «больше игры, меньше музея». Снаппи-сканы, акцент на ощущении
  полёта; лор — лёгкий бонус.
- Масштаб: реальные пропорции EVE + **управляемый варп как в Elite supercruise** (с
  разгоном/торможением, не телепорт). Инерция = аркадный дрейф + тормоз.
- В MVP хотели: синтез-звук + мини-карта/радар. НЕ брали: фото-режим, журнал/именование,
  топливо-выживание.
- Работать малыми, по отдельности проверяемыми, отдельно коммиченными шагами; проверять в
  браузере. Общение на русском.
